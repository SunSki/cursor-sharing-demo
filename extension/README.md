# LiveCursors Chrome 拡張機能

拡張機能を入れた人同士が**同じURLを開いているとき**、お互いのカーソルを
リアルタイムで表示します。サイト側に `<script>` を追加する必要はありません。

## 仕組み

- content script が全ページに `socket.io` と `livecursors.js`（拡張機能に同梱）
  を注入します。
- ルームは現在のページの `origin + pathname`。同じページを見ている拡張機能
  ユーザーが同じルームに入ります。
- カーソルは Render 上のリレーサーバー
  (`https://cursor-sharing-demo.onrender.com`) 経由で中継されます。
- カーソル位置は「どのDOM要素の中の相対位置か」で送るので、画面サイズが
  違っても同じ要素を指せます。

## インストール（開発者モードで読み込み）

1. Chrome で `chrome://extensions` を開く
2. 右上の「**デベロッパーモード**」をオンにする
3. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
4. この `extension/` フォルダを選択

ツールバーにアイコンが出れば完了です。

## 使い方

1. 任意のWebページを開く（同じページを別の人／別ウィンドウでも開く）
2. マウスを動かすと、相手の画面にあなたのカーソルが表示される
3. 画面右下に「LiveCursors 〇人」のバッジが出れば接続中

ツールバーのアイコンをクリックすると：
- **ON/OFF の切り替え**
- **表示名の設定**（空欄ならランダム）

設定変更後はページを再読み込みすると確実に反映されます。

## テスト方法（自分ひとりで確認）

1. 適当なページ（例: `https://example.com`）を**2つのウィンドウ**で開く
2. 両方とも拡張機能が有効な状態にする
3. 片方でマウスを動かすと、もう片方に色付きカーソルと名前が表示される

> 別ウィンドウでも別タブでも、同じURLなら同室になります。

## 制約・注意

- **相手も同じ拡張機能を入れている必要があります。**
- ルームは `origin + pathname` 単位。クエリ文字列やハッシュは無視します。
- Render 無料プランはスリープするため、最初の接続に数十秒かかることがあります。
- カーソル位置と表示名がリレーサーバーに送信されます（同じURLの利用者に共有）。
- `http(s)` ページのみで動作します（`chrome://` などでは動きません）。

## 構成

```
extension/
├── manifest.json          MV3 マニフェスト
├── content.js             ページに注入される起動スクリプト
├── popup.html / popup.js  ON/OFF・表示名の設定UI
└── lib/
    ├── socket.io.min.js   Socket.IO クライアント（v4.7.5・同梱）
    └── livecursors.js     @livecursors/client のコピー
```

## リレーサーバーを自分のものに変える

`content.js` の先頭 `RELAY` を自分のサーバーURLに変更してください。
そのサーバーは `attachCursors(httpServer, { path: '/livecursors', cors: { origin: '*' } })`
で起動している必要があります（リポジトリルートの `server.js` がその例）。
