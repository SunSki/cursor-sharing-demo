# DotSync Chrome 拡張機能

**ルームコードを共有した人同士**が、同じURLを開いているときにお互いのドット（位置マーカー）を
リアルタイムで表示します。サイト側に `<script>` を追加する必要はありません。

## 仕組み

- content script が全ページに `socket.io` と `livecursors.js`（拡張機能に同梱）
  を注入します。
- ルームは `origin + pathname + ":" + ルームコード`。**同じコード × 同じURL** の
  人だけが同じルームに入ります。
- カーソルは Render 上のリレーサーバー
  (`https://cursor-sharing-demo.onrender.com`) 経由で中継されます。
- カーソル位置は「どのDOM要素の中の相対位置か」で送るので、画面サイズが
  違っても同じ要素を指せます。

## インストール（開発者モードで読み込み）

1. Chrome で `chrome://extensions` を開く
2. 右上の「**デベロッパーモード**」をオンにする
3. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
4. この `extension/` フォルダを選択

## 使い方

1. ツールバーのアイコンをクリックしてポップアップを開く
2. **ルームコード**を入力（「生成」ボタンでランダム作成も可・最大20文字）
3. 同じコードを、見せ合いたい相手に共有する
4. 全員が同じコードを入力し、同じページを開くとカーソルが見える

- **ルームコードが空欄のときは機能OFF**（接続しません）
- 表示名は任意（空欄ならランダム）
- 設定は `chrome.storage.sync` に保存され、Chromeにサインインしていれば
  デバイス間で同期されます

## テスト方法（自分ひとりで確認）

1. ポップアップでルームコードを設定（例:「生成」ボタン）
2. 適当なページ（例: `https://example.com`）を**2つのウィンドウ**で開く
3. 片方でマウスを動かすと、もう片方に色付きドットと名前が表示される
4. 相手が画面外に出ると、画面の上端／下端に ▲/▼ + 名前 で方向が表示される

## サーバー負荷を抑える仕組み

多人数でも軽く動くよう、クライアント側で以下を行っています。

- **1人のときは送信しない**（同室に他の人がいるときだけ送信）
- **タブが非表示のとき送信停止**
- **5秒間マウスが止まったら送信停止**（動かすと再開）
- 送信頻度 20fps（50ms）、ペイロードは短縮キー＋座標を丸めて軽量化
- サーバー側は WebSocket 優先・メッセージ圧縮OFF

## 制約・注意

- **相手も同じ拡張機能を入れ、同じルームコードを設定する必要があります。**
- ルームコードを知っている人は誰でも同じルームに入れます（招待制ではなく合言葉方式）。
- ルームは `origin + pathname` 単位。クエリ文字列やハッシュは無視します。
- Render 無料プランはスリープするため、最初の接続に数十秒かかることがあります。
- カーソル位置と表示名がリレーサーバーに送信されます（同ルームの利用者に共有）。
- `http(s)` ページのみで動作します（`chrome://` などでは動きません）。

## 構成

```
extension/
├── manifest.json          MV3 マニフェスト
├── content.js             ルームコードがあれば起動／変更を監視
├── popup.html / popup.js   ルームコード・表示名の設定UI（コード生成つき）
└── lib/
    ├── socket.io.min.js   Socket.IO クライアント（v4.7.5・同梱）
    └── livecursors.js     @livecursors/client のコピー
```

## リレーサーバーを自分のものに変える

`content.js` の先頭 `RELAY` を自分のサーバーURLに変更してください。
そのサーバーは `attachCursors(httpServer, { path: '/livecursors', cors: { origin: '*' } })`
で起動している必要があります（リポジトリルートの `server.js` がその例）。
